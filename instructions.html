Running C Code

A Beginner's Guide for Linux

Step 1: Open Your Terminal

The terminal is your command center in Linux. It's a powerful tool where you can run commands to control your computer. You can usually find it in your applications menu or by pressing Ctrl+Alt+T.

Step 2: Create a Workspace

It's a good practice to keep your projects organized. Let's create a dedicated folder for your C programs. First, navigate to your Desktop and create a new directory called c-programs.

Use the cd (change directory) command to move into your Desktop:

cd Desktop

Now, use the mkdir (make directory) command to create the new folder:

mkdir c-programs

Finally, move into your new workspace:

cd c-programs

IMPORTANT: You must always be inside this folder when you work on your C files for this lab!

Step 3: Create Your C File

Now that you're in the right directory, you can create your first C source file. We'll call it program1.c. Use the touch command to create an empty file.

touch program1.c

You can type ls to list the files and see that program1.c has been created.

Step 4: Write the Code

The touch command only creates an empty file. You need a text editor to write your code. You can use a simple terminal editor like nano or a graphical editor like VS Code.

For example, to use nano, you would type:

nano program1.c

Now, copy the code for one of the problems into this file. After you're done, save the file and exit the editor (in nano, you press Ctrl+X, then Y, then Enter).

Step 5: Compile with GCC

Your C code is human-readable, but the computer needs it in a format called "machine code". The process of converting your code is called **compiling**. We use a compiler named GCC (GNU Compiler Collection).

To compile program1.c, run the following command:

gcc program1.c -o program1

Let's break this down:

  • gcc: This is the command to call the compiler.
  • program1.c: This is your source code file (the input).
  • -o program1: The -o is a flag that means "output file". We are telling GCC to name the compiled, runnable program program1 (the output). If you don't use this, it will be named a.out by default.

If there are no errors in your code, the command will complete silently. If you type ls now, you will see a new file: program1 (it might be a different color!). This is your executable program.

Step 6: Run Your Program!

Now it's time to run your compiled program. Use the following command:

./program1

What does ./ mean? It tells the terminal to look for the program in the **current directory**. For security reasons, Linux doesn't look in the current directory for commands by default, so you have to be explicit.

Your program should now run, and you'll see its output in the terminal!

What If I Close The Terminal?

Don't worry! Your files are saved. When you open a new terminal, it usually starts in your home directory (~). You just need to navigate back to your workspace before you can compile or run your files again.

Just use the cd command from Step 2 to get back into your folder:

cd Desktop/c-programs

Once you are in the correct directory, you can use gcc and ./ commands just like before.
